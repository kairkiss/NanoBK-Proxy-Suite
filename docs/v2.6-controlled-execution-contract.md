# NanoBK v2.6 Controlled Real Execution Contract

v2.6 starts the controlled real execution layer for NanoBK production setup.
v2.5 remains sealed as the read-only production CLI readiness layer.

This document defines the execution contract only. v2.6.0 does not perform
real DNS changes, Worker deployment, certificate issuance, VPS installation,
token rotation, protocol key rotation, or service reloads.

## Product Contract

- One-command install remains install-only.
- Install must not automatically deploy production services.
- `nanobk` remains the product entry point.
- `nanobk setup production execute-plan` shows the future real execution plan.
- `nanobk beginner production execute-plan` is the beginner-facing alias.
- Commands must use beginner wording by default.
- Dangerous operations must stay controlled by explicit policy.
- Raw secrets and internal Cloudflare identifiers must not be printed.

## Action Types

| Action type | Product label | Meaning |
| --- | --- | --- |
| `read_only` | 只读检查 | Inspect current readiness without changing anything. |
| `local_write` | 本地配置保存 | Write local NanoBK plan/profile files only. |
| `cloudflare_mutation` | 域名指向 | Change Cloudflare DNS or routing state. |
| `cert_issue` | HTTPS 安全证书 | Request or install a production HTTPS certificate. |
| `worker_deploy` | 订阅服务入口 | Deploy or update the Cloudflare Worker subscription entry. |
| `vps_install` | 代理服务安装 | Install or update VPS proxy service files. |
| `token_rotate` | 重新生成订阅密钥 | Rotate subscription access tokens. |
| `protocol_rotate` | 更新代理通道密钥 | Rotate proxy protocol keys. |
| `service_reload` | 服务重载 | Reload or restart production services after changes. |

## Execution Policies

| Policy | Meaning |
| --- | --- |
| `preview_only` | Show the plan only. Never execute the operation. |
| `manual_only` | The user must perform the operation manually. |
| `confirm_required` | The user must confirm before the operation can run. |
| `exact_gate_required` | The user must provide the exact authoritative gate phrase. |

## v2.6.0 Command Scope

`nanobk setup production execute-plan` and
`nanobk beginner production execute-plan` are preview-only.

They may show which operations are planned for v2.6, but they must not execute
any real production operation.

They may read existing v2.5 readiness/deploy-plan/gate structures in-process.
They must not shell out to NanoBK commands or call external programs.

## v2.6.0 Safety Contract

- `mutation=false`
- `dangerous_actions_executed=false`
- `execution_enabled=false`
- `policy=preview_only`
- `safety=read_only`
- All dangerous future actions are `manual_only=true`.
- All dangerous future actions are `implemented_now=false`.
- All dangerous future actions are `will_modify=true`.

The v2.3 exact confirmation gates remain authoritative for dangerous DNS,
certificate, and token actions.

## Future Execution Order

The long-term guided flow is:

1. 域名指向
2. 订阅服务入口
3. HTTPS 安全证书
4. 代理服务安装
5. 重新生成订阅密钥
6. 更新代理通道密钥

v2.6.0 only displays this order and the safety policy for each step.
