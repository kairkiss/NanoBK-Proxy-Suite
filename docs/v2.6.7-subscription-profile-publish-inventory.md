# v2.6.7 Subscription/Profile Publish Inventory

Baseline: `5f88dbe test: de-recursive v2.6 regression harness`

This inventory grounds the v2.6.7 subscription publish wrapper in the existing
repository assets. It does not change the legacy Worker, Cloudflare installer,
VPS installer, token rotation scripts, or systemd files.

## Legacy Publish Assets

- Legacy profile upload entrypoint: `installer/install-cloudflare.sh`
  function `upload_profile()`.
- Legacy profile path: `/etc/nanobk/profile.current.json`.
- Legacy upload method: HTTP `POST` to the Worker admin update endpoint with
  `Authorization: Bearer ...` and JSON body from the profile file.
- Legacy verification path: `verify_nanok()` reads the Worker admin current
  endpoint, then checks for the four protocol sections.
- CLI admin env installer: `bin/nanobk cf install-admin-env` copies admin
  values from `.cloudflare.local.env` into `/root/.nanok-cf-admin.env`; it does
  not publish the profile.

## Worker Contract

- Primary nanok Worker source: `workers/nanok/src/index.js`.
- Profile KV key: `profile:main`.
- KV binding name: `SUB_STORE`.
- Default subscription path: `/jb`.
- Default admin update path: `/admin/update`.
- Default admin current path: `/admin/current`.
- Required admin auth: bearer token from `ADMIN_TOKEN`.
- Admin update handler: `updateProfile()` validates the profile, writes the
  cleaned profile to KV, and returns a short status response.
- Admin current handler returns the current raw profile and must not be printed
  by the v2.6.7 wrapper.

## Admin Env Files And Fields

Detected local env files:

- `/root/.nanok-cf-admin.env`
- `.cloudflare.local.env`
- `.nanob.local.env`

Required fields for profile publish:

- `ADMIN_UPDATE_URL`
- `ADMIN_CURRENT_URL`
- `ADMIN_TOKEN`

Related fields that must remain hidden:

- `SUB_TOKEN`
- `SUB_PATH`
- Cloudflare account/zone/KV identifiers

The wrapper reads env files with a small parser and never sources them.

## VPS Profile Requirements

The publish wrapper checks `/etc/nanobk/profile.current.json` before any publish
attempt. The profile must be valid JSON and contain all four protocol sections:

- `hy2`
- `tuic`
- `reality`
- `trojan`

The profile may contain UUIDs, passwords, public keys, private keys, and other
secrets. v2.6.7 only emits a redacted summary: existence, completeness, protocol
names, and optional non-secret metadata.

## New Wrapper Calls

New v2.6.7 wrapper:

- `lib/nanobk_production_subscription_publish.py`

The wrapper calls:

- `nanobk_domain_selection.load_selected_domain()` for the selected domain.
- `nanobk_setup_profile.load_profile()` as a legacy selected-domain fallback.
- A local safe env parser for `/root/.nanok-cf-admin.env`,
  `.cloudflare.local.env`, and `.nanob.local.env`.
- A guarded HTTP admin-update adapter only after exact confirmation and
  `NANOBK_ALLOW_REAL_PROFILE_PUBLISH=1`.

The wrapper does not call `installer/install-cloudflare.sh` because that script
also owns Worker deploy, secret setup, verification, and other Cloudflare setup
steps. v2.6.7 uses only the discovered Worker admin update contract.

## Files Explicitly Not Touched

- `workers/*`
- `installer/install-cloudflare.sh`
- `installer/install-vps.sh`
- `vps/*`
- token rotation scripts
- systemd files
- DNS, Worker deploy, cert issue, and VPS install wrappers

## Safety Risks

- Raw `profile.current.json` contains protocol secrets.
- Admin and subscription tokens must never be printed.
- Admin update/current URLs may include sensitive route information and must not
  be printed.
- Worker current responses contain the raw profile and must not be printed.
- A real publish may update the live subscription Worker profile.
- Worker API contract drift could make a publish fail after the request is sent.

## Guarded And Fake Behavior

- Default commands are read-only.
- Real publish requires the exact phrase:
  `I UNDERSTAND NANOBK WILL PUBLISH SUBSCRIPTION PROFILE`.
- Real publish also requires `NANOBK_ALLOW_REAL_PROFILE_PUBLISH=1`.
- Automated tests use fake publish hooks and never call the network.
- If admin env or profile readiness is incomplete, the wrapper blocks before
  any publish attempt.
