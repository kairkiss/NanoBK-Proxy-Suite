# v2.2 to v2.3 Handoff

## Current Final Command Surface

### CLI

```
nanobk home [--json]
nanobk setup status [--json]
nanobk setup wizard [--zone DOMAIN] [--api-env PATH] [--nodes proxy,web] [--yes] [--json]
nanobk setup dns [--zone DOMAIN] [--api-env PATH] [--nodes proxy,web] [--json]
nanobk setup profile save --zone DOMAIN --api-env PATH [--nodes proxy,web] [--json]
nanobk setup profile show [--json]
nanobk setup profile clear [--json]
nanobk cf dns availability summary --zone DOMAIN --api-env PATH [--nodes proxy,web] [--json]
nanobk cf dns plan-generator --zone DOMAIN --api-env PATH [--json]
nanobk cf dns create-preflight --zone DOMAIN --api-env PATH [--json]
nanobk cf dns owner-smoke-create --zone DOMAIN --api-env PATH --label ... --type ... --content ... [--json]
nanobk cf zones list --api-env PATH [--json]
nanobk vps ip detect [--json]
```

### Web

```
GET /api/home         — read-only home/status JSON (requires login)
GET /api/status       — read-only system status JSON (requires login)
```

### Bot

```
/home                 — setup home summary (owner only)
/setup_status         — setup status alias (owner only)
/status               — system status summary (owner only)
/status_json          — system status JSON (owner + advanced mode)
```

## Current Safe Data Flow

```
setup profile save
  -> setup dns (reads profile, runs IP detect + availability + plan)
  -> explanation (adds next actions, fix hints)
  -> home (reads profile, runs setup, adds explanation)
  -> Web /api/home (calls nanobk home --json, redacts)
  -> Bot /home (calls get_home_text adapter, redacts)
```

All paths are read-only. No DNS mutation occurs in this flow.

## v2.3 Recommended Focus

### Web Dashboard

- Real UI card for home/status on dashboard page
- Setup status card with next action buttons
- i18n polish for home/setup/help text
- Richer setup troubleshooting guidance

### Bot Integration

- Home button in control center menu
- Setup status button in control center menu
- i18n polish for home/setup/help text
- Inline keyboard for next actions

### CLI Polish

- Richer setup troubleshooting guidance
- Better error messages for common failures
- Progress indicators for long-running checks

### Owner-Controlled DNS Create

- Design doc for controlled create flow
- Still gated behind owner approval + confirmation
- Disposable-first, cleanup-required
- No production proxy/web auto-apply

## Safety Boundaries to Preserve

- No DNS mutation without explicit owner approval
- No production proxy/web DNS creation without separate design doc
- No Bot/Web live create without separate review
- No Full Wizard auto-apply DNS
- Token/credential/profile path/zone ID/record ID/raw API never printed
- Web login protection never lowered
- Bot owner gate never bypassed
