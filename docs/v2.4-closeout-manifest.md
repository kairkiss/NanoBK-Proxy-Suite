## Known Non-Blocking Notes

### 1. Safety grep false positives

Filtered hard grep may match test self-report lines such as:

- `[OK] no /dns_records`
- `[OK] no CF_ZONE_ID`
- `[OK] no api_env_path`

These are not secret leaks. They are test self-report labels confirming the absence of secrets. Accepted after manual review.

### 2. DNS gate wording polish

Current DNS friendly gate wording includes:

> "如果已有同名记录，可能会被覆盖"

Actual v2.4.5 wrapper is preview-only and does not execute DNS apply. Future wording can be improved to:

> "如果已有同名记录，系统会要求你处理冲突，不会自动覆盖。"

This is non-blocking. Should not be changed in v2.4.7 unless explicitly approved.

### 3. v2.2.55 old JSON contract

v2.4 safe JSON is now the default for `home --json`. Old format remains available via:

```
nanobk home --legacy-json
```

Do not treat old v2.2.55 `home --json` expectations as v2.4 blocker.

### 4. Long regression opt-in

Legacy / environment-dependent long regression is opt-in:

```
NANOBK_RUN_LONG_REGRESSION=1
```

Default v2.4 tests do not run long recursive acceptance chains. This prevents timeout on fresh VPS clone.
