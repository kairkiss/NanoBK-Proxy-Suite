# NanoBK v2.4 Beginner Production Setup UX

## Status

- Planning / scope lock
- No runtime behavior change in v2.4.0
- Built on v2.3 CLI Automation Gate

## Product Goal

NanoBK v2.4 的目标是让用户安装后输入 `nanobk`，进入一个清楚、好看的中文设置中心。它会帮用户检查 Cloudflare、域名、VPS IP、DNS、证书和订阅，但危险修改仍然需要明确确认。

v2.4 不是重写 v2.3。v2.4 是让普通用户更容易使用 v2.3。

## Non-Goals

- v2.4.0 不创建 DNS
- v2.4.0 不申请证书
- v2.4.0 不轮换 token
- v2.4.0 不修改 Worker
- v2.4.0 不 reload/restart 服务
- v2.4.0 不发布 release/tag

## UX Principles

1. **小白优先** — 不假设用户知道 Cloudflare、DNS、证书是什么
2. **默认中文** — 所有界面和提示使用简体中文
3. **默认隐藏工程日志** — 不显示 raw API response、zone id、record id
4. **默认只读** — 所有检查和规划不修改任何外部系统
5. **危险动作必须明确确认** — 继承 v2.3 exact confirmation phrase 机制
6. **不泄露 secret** — token、private key、subscription URL 永远不输出
7. **失败时给下一步建议** — 不只说"失败了"，告诉用户该怎么做
8. **不让用户看到 raw Cloudflare API / zone id / record id** — 只显示域名和状态

## Beginner Flow

1. **欢迎页** — 介绍 NanoBK 是什么，告诉用户接下来会做什么
2. **Cloudflare 连接** — 引导用户粘贴 API token，只读验证
3. **自动发现域名** — 列出 Cloudflare 账户下的域名
4. **用户选择域名** — 如果多个域名，让用户选一个
5. **自动检测 VPS IPv4/IPv6** — 显示 masked IP
6. **自动规划 proxy/web 子域名** — 建议 `proxy.用户域名` 和 `web.用户域名`
7. **检测子域名是否可用** — 检查 Cloudflare DNS 是否已有记录
8. **冲突时请求用户输入自定义子域名** — 不覆盖、不删除，让用户换名字
9. **显示即将做什么** — 列出所有待执行操作，全部只读
10. **进入 DNS gate** — 显示 plan，要求 exact confirmation 才能 apply
11. **进入 cert gate** — 显示 preflight，要求 exact confirmation 才能 issue
12. **进入 token gate** — 显示 rotation plan，要求 exact confirmation 才能 rotate
13. **输出最终状态** — 汇总所有步骤结果

## Friendly Language Mapping

| 工程术语 | 小白语言 |
|---------|---------|
| Cloudflare Zone | 你的域名 |
| DNS Record | 域名指向 |
| A/AAAA Record | 把域名指向你的服务器 IP |
| Certificate | HTTPS 安全证书 |
| Token Rotation | 重新生成订阅密钥 |
| Mutation | 实际修改 |
| Apply | 确认执行 |
| Worker | 订阅服务入口 |
| Preflight | 预检查 |
| Gate | 安全门禁 |

## Safety Model

继承 v2.3 安全底线：

- **read-only by default** — 所有检查和规划不修改外部系统
- **exact confirmation phrase required** — DNS apply、cert issue、token rotate 都需要精确确认短语
- **production Worker protected** — `nanok`、`nanob`、`nanok.biankai314.uk`、`nanob.biankai314.uk` 默认阻止
- **no raw secret output** — token、zone id、record id、raw API response 永远不输出
- **no automatic DNS/cert/token/Worker mutation** — 不自动创建 DNS、不自动申请证书、不自动轮换 token、不自动修改 Worker
- **no reload/restart without explicit owner approval** — 不自动 reload/restart 服务
