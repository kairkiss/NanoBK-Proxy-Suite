# Key Rotation Guide

One-command credential rotation for all four proxy services.

## Quick Start

```bash
# Rotate all protocols
sudo bash vps/scripts/rotate-keys.sh

# Rotate single protocol
sudo bash vps/scripts/rotate-keys.sh --protocol hy2
sudo bash vps/scripts/rotate-keys.sh --protocol reality

# Via unified CLI
nanobk rotate all
nanobk rotate hy2
nanobk rotate tuic
nanobk rotate reality
nanobk rotate trojan
```

## What It Does

1. **Loads** config from `/etc/nanobk/config.env` and `secrets.private.env`
2. **Loads** Cloudflare admin tokens from `/root/.nanok-cf-admin.env`
3. **Generates** new credentials (staged, not yet committed)
4. **Backs up** all configs to `${INSTALL_DIR}/backups/rotate-YYYYmmdd-HHMMSS/`
5. **Patches** local service configs using structured Python parsers
6. **Generates** new `profile.current.json` (saves old as `profile.previous.json`)
7. **Validates** JSON configs (xray -test if available)
8. **Restarts** services one by one
9. **Runs** healthcheck
10. **Syncs** new profile to Cloudflare via admin API
11. **Verifies** Cloudflare profile matches new credentials
12. **Commits** new secrets to `secrets.private.env` (mode 600)
13. **On failure**: rolls back all local configs

## Configuration Sources

| File | Purpose |
|------|---------|
| `/etc/nanobk/config.env` | VPS paths, domain, IP, service names |
| `/etc/nanobk/secrets.private.env` | Current credentials (read + written) |
| `/root/.nanok-cf-admin.env` | Cloudflare admin tokens |

The `.cloudflare.local.env` generated by `install-cloudflare.sh` can be used as the CF admin env. The script auto-derives `ADMIN_CURRENT_URL` and `ADMIN_UPDATE_URL` from `NANOK_ROUTE_URL` + paths.

## Single-Protocol Rotation

Rotate only one protocol while keeping others unchanged:

```bash
# Only HY2 password
sudo bash vps/scripts/rotate-keys.sh --protocol hy2

# Only TUIC UUID + password
sudo bash vps/scripts/rotate-keys.sh --protocol tuic

# Only Reality keys
sudo bash vps/scripts/rotate-keys.sh --protocol reality

# Only Trojan password
sudo bash vps/scripts/rotate-keys.sh --protocol trojan
```

Single-protocol rotation:
- Only generates new credentials for the selected protocol
- Still generates a complete `profile.current.json` (all four protocols)
- Still patches only the selected protocol's service config
- Still restarts only the selected protocol's service
- Cloudflare verification checks only the rotated protocol's fields

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `--protocol` | `all` | Protocol to rotate: `all\|hy2\|tuic\|reality\|trojan` |
| `--dry-run` | off | Print actions without modifying anything |
| `--yes` | off | Non-interactive mode |
| `--config-dir` | `/etc/nanobk` | Config directory |
| `--install-dir` | `/opt/nanobk` | Install directory |
| `--cf-admin-env` | `/root/.nanok-cf-admin.env` | Cloudflare admin env file |
| `--skip-cloudflare` | off | Local-only rotation (no CF sync) |
| `--skip-services` | off | No service restart or port check |
| `--allow-placeholder-reality` | off | Allow placeholder Reality key when xray missing (TEST ONLY) |
| `--force` | off | Skip confirmation |

**`--allow-placeholder-reality`** is for offline testing only. Never use in production. Without xray, Reality keypair cannot be generated. If xray is missing and this flag is not set, rotation fails with a clear error.

## Test Mode

Test rotation without real services or Cloudflare:

```bash
# First generate test configs
bash tests/render-install-vps.sh

# Then run rotation on the test configs
bash vps/scripts/rotate-keys.sh --yes \
  --config-dir /tmp/nanobk-render-test/etc/nanobk \
  --install-dir /tmp/nanobk-render-test/opt/nanobk \
  --skip-services --skip-cloudflare \
  --allow-placeholder-reality
```

Or use the combined test (includes rollback failure test):

```bash
bash tests/rotate-render-only.sh
```

## Backup Files

Each protocol's config is backed up with a unique name (no collision):

| Protocol | Backup filename |
|----------|----------------|
| HY2 | `hy2.config.yaml.bak` |
| TUIC | `tuic.config.json.bak` |
| Reality | `reality.config.json.bak` |
| Trojan | `trojan.config.json.bak` |

This ensures rollback restores each protocol's own config correctly.

## Credentials Rotated

| Service | Rotated Fields |
|---------|---------------|
| HY2 / Hysteria2 | password |
| TUIC v5 | UUID, password |
| VLESS Reality | client UUID, private key, public key, short ID |
| Trojan TLS | password |

**NOT rotated**: domains, ports, certificates, SSH, firewall, Cloudflare DNS/Tunnel/Access.

## Rollback

Automatic on failure. Manual rollback:

```bash
# Find backup
ls -lt /opt/nanobk/backups/rotate-*/ | head

# Restore from backup
cp /opt/nanobk/backups/rotate-YYYYMMDD-HHMMSS/* /etc/nanobk/

# Restart services
systemctl restart hysteria-server.service
systemctl restart tuic-v5-9443.service
systemctl restart xray-reality-8443.service
systemctl restart xray-trojan-2443.service
```

If Cloudflare was already updated, re-run rotation or manually POST old profile:

```bash
curl -X POST https://YOUR_WORKER/admin/update \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  --data-binary @/opt/nanobk/backups/rotate-YYYYMMDD-HHMMSS/profile.current.json
```

## Security

- Full credentials only in `secrets.private.env` (mode 600)
- Rotation record at `backups/rotate-latest.private.md` (fingerprints only, mode 600)
- Reality private key NEVER enters profile JSON
- Subscription URL never changes — clients refresh to get new keys
