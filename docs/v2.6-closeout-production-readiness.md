# v2.6 Closeout: Production Readiness

## Milestone Summary

v2.6 turns NanoBK from separate production wrappers into a guided, read-only-by-default production flow.

The milestone connects the owner-facing path across Cloudflare, domain selection, DNS, Worker, certificate, VPS four-protocol install, subscription profile publish, owner review, and guided production flow. Each dangerous operation remains behind its own controlled wrapper, exact confirmation phrase, and environment guard where real execution exists.

The recommended owner entrypoints are:

```bash
nanobk beginner production
nanobk setup production guide
```

These entrypoints are review and guidance surfaces. They do not perform hidden infrastructure changes.

## Safety Doctrine

- Default behavior is read-only.
- Dangerous execution requires exact phrase gates.
- Real execution paths require explicit environment guards where implemented.
- NanoBK must not perform hidden mutation.
- NanoBK must not print raw secrets.
- Unknown state blocks instead of guessing.
- Partial install blocks and routes to `repair_or_review`.
- Failure after a controlled action routes to `repair_or_review` with redacted summaries.

## Dangerous Actions

These actions are not run by the production guide or owner review:

- DNS apply
- Worker deploy
- cert issue
- VPS install
- subscription profile publish
- token rotation
- protocol key rotation
- service restart/reload

Each action must stay in its own controlled command surface.

## Release Blockers

The v2.6 milestone is closeout-ready for guided flow preparation, but not a final production release. These blockers remain visible:

- clean VPS full real install not yet validated
- live profile publish not yet validated

Do not mark the product as fully production-complete until both blockers are cleared.

## v2.7 Handoff

v2.7 is for real clean VPS beginner acceptance, not more internal fake-only work.

The next milestone should validate a real beginner path on a clean/disposable VPS using beginner-facing commands only, preserving the v2.6 safety doctrine and proving the remaining release blockers.
