## Purpose

This runbook describes the operator-only disposable DNS create smoke path.

It does **not** enable production proxy/web DNS creation.

Production proxy/web DNS creation remains blocked.

## Current Allowed Scope

### Allowed Only For

- Disposable smoke labels starting with `nanobk-smoke-`
- A / AAAA / TXT records only
- Owner-approved smoke
- Cleanup-required smoke
- Exact confirmation phrase
- Post-check
- Cleanup verification
- Leftover check
- Scanner redaction check

### Not Allowed For

- `proxy`
- `web`
- `www`
- `@`
- `*`
- `api`
- `cdn`
- `mail`
- `smtp`
- `imap`
- Production hostnames
- Any non-`nanobk-smoke-*` label

## Required Gates Before Mutation

1. Clean git status
2. Expected HEAD
3. Credential file permission 0400 or 0600
4. Owner explicitly approves (`--owner-approve`)
5. Exact confirmation phrase provided (`--confirm-disposable-smoke`)
6. `--cleanup` provided
7. Label starts with `nanobk-smoke-`
8. Record type is A / AAAA / TXT
9. Content passes public routable validation
10. Pre-check confirms no existing record

## Required Operation Order

1. GET pre-check â€” confirm target FQDN does not exist
2. POST create â€” create the disposable record
3. GET post-check â€” confirm created record matches name/type/content
4. DELETE cleanup â€” delete only the record created in this run
5. GET cleanup verification â€” confirm no matching target record remains
6. Read-only leftover availability check â€” confirm no persistent change
7. Scanner redaction check â€” confirm no secrets leaked
8. Archive result â€” document safe booleans only

## Failure Handling

### Post-check Failure

If create succeeds but post-check fails:

- Attempt cleanup
- Mark `status=post_check_failed`
- Exit non-zero
- Do not print record ID
- Require manual dashboard verification if uncertainty remains

### Cleanup Failure

If cleanup fails:

- Mark `status=cleanup_failed`
- Exit non-zero
- Do not print record ID
- Require manual Cloudflare dashboard verification
- Do not retry destructive cleanup blindly
- Do not touch unrelated records

### Leftover Found

If leftover check finds matching record:

- Stop
- Do not proceed to production features
- Require manual cleanup review
- Archive as BLOCKED / NEEDS POLISH

## Scanner Rules

Reference v2.2.47 behavior.

### Safe

- `raw_api_response_printed=false`
- `"raw_api_response_printed": false`
- `Raw API response printed: no`
- `api.cloudflare.com printed: no`
- `/zones/ printed: no`
- `/dns_records printed: no`
- Forbidden-list documentation text

### Leaks

- Real API URL (`https://api.cloudflare.com/client/v4/...`)
- Authorization Bearer header
- Token env values (`CF_API_TOKEN=...`)
- Credential path
- Zone ID
- Record ID
- Raw API response body
- Protocol links (`vless://`, `trojan://`, `hysteria2://`, `tuic://`)
- Subscription URLs
- Private keys
