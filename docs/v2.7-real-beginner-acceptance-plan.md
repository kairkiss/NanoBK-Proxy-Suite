# v2.7 Real Beginner Acceptance Plan

## Goal

v2.7 validates that a real beginner tester can start on a clean/disposable VPS and follow only beginner-facing NanoBK commands until they reach a usable subscription.

The acceptance must be based on current repository behavior and real command outputs. It must not depend on developer memory.

## Required Validation Scope

v2.7 must validate:

1. clean VPS baseline
2. no preexisting `/etc/nanobk`
3. no preexisting four protocol services
4. Cloudflare/domain setup visibility
5. DNS dry-run and gated apply
6. Worker dry-run and gated deploy
7. cert dry-run and gated issue
8. VPS four-protocol dry-run and gated install
9. profile generation
10. subscription publish dry-run and optional live publish
11. owner review
12. final subscription import
13. no secret leakage
14. recovery from partial failure
15. user comprehension

## Required v2.7 Tests

| Test | Purpose |
|---|---|
| T29-CLEAN | Clean VPS full install with the real legacy VPS installer adapter |
| T30-LIVE | Optional live profile publish to the existing subscription Worker |
| T33-BEGINNER | Full guided flow using beginner-facing commands only |
| T34-FAILURE | Recovery and partial install handling |
| T35-LEAK | End-to-end leak audit across text, JSON, logs, and docs |
| T36-DOCS | Beginner comprehension review for the documented path |

## Expected Beginner Path

The tester should begin with:

```bash
nanobk beginner production
```

Then follow only the shown next commands. Dry-runs should come before every dangerous step. Real execution should require the exact phrase and environment guard owned by the individual controlled wrapper.

## Acceptance Rules

- No dangerous action may run from the guide itself.
- Every real operation must be owner-confirmed.
- Every output must stay redacted.
- Partial or unknown state must block.
- A failed real operation must return a repair or review path.
- The final result must be a usable subscription verified by import or equivalent owner-approved validation.
