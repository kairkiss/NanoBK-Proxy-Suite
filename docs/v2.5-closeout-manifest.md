# v2.5 Closeout Manifest — Production CLI Readiness Layer

## Scope

### What v2.5 adds

v2.5 provides a full read-only production setup product flow for NanoBK Proxy Suite. It covers DNS, Worker, certificate, token/rotation, overview, next-step, preflight, deploy-plan, and gates — all as safe, non-destructive CLI commands.

### What v2.5 does not do

- Does not execute real DNS apply
- Does not issue certificates
- Does not rotate tokens
- Does not rotate protocol keys
- Does not deploy Workers
- Does not install VPS protocols
- Does not restart/reload services
- Does not create tags or releases
- Does not output raw secrets

### Read-only commands

All `nanobk setup production` and `nanobk beginner production` subcommands are read-only by default:

- `nanobk setup production` / `status` / `plan`
- `nanobk setup production actions`
- `nanobk setup production dns`
- `nanobk setup production worker`
- `nanobk setup production cert`
- `nanobk setup production rotate`
- `nanobk setup production overview`
- `nanobk setup production next`
- `nanobk setup production preflight`
- `nanobk setup production deploy-plan`
- `nanobk setup production gates`

### Manual-only commands

Dangerous operations remain manual-only and require exact phrase confirmation:

- `nanobk setup dns apply`
- `nanobk setup cert issue`
- `nanobk setup token rotate`

## Safety contract

- No DNS apply by default
- No cert issue by default
- No token rotate by default
- No protocol key rotation by default
- No Worker deploy by default
- No VPS install by default
- No service restart/reload by default
- No tag/release by default
- No raw secret output

## Exact gates

Authoritative v2.3 confirmation phrases:

- **DNS:** `I UNDERSTAND NANOBK WILL CREATE CLOUDFLARE DNS RECORDS`
- **Cert:** `I UNDERSTAND NANOBK WILL REQUEST TLS CERTIFICATES`
- **Token:** `I UNDERSTAND NANOBK WILL ROTATE SUBSCRIPTION TOKENS`
