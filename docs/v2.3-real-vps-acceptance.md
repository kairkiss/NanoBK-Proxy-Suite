# v2.3 Real VPS Acceptance

## Scope

v2.3 is CLI automation gate and setup orchestration. It provides read-only
planning and gated execution for Cloudflare DNS, ACME certificates, and
subscription token rotation.

## Implemented

- **v2.3.0** CLI Cloudflare/DNS/Cert/Token Automation Scope Lock
- **v2.3.1** Brand CLI Console Shell
- **v2.3.2** Cloudflare Account/Zone Onboarding
- **v2.3.3** Domain/IP/Subdomain Planner
- **v2.3.4** DNS Apply Engine with Confirmation
- **v2.3.5** Certificate Automation Preflight
- **v2.3.6** Certificate Issue Gate
- **v2.3.7** Subscription Token Rotation Gate
- **v2.3.8** Full CLI Setup Flow Integration
- **v2.3.9** Real VPS Acceptance & Closeout

## Safety Model

- **Read-only by default**: All planning/preflight commands do not mutate DNS, certificates, tokens, or Workers.
- **Exact confirmation phrases**: Dangerous actions require explicit `--apply`/`--issue`/`--rotate` plus an exact `--confirm` phrase.
- **No production Worker mutation**: `nanok`, `nanob`, `nanok.biankai314.uk`, `nanob.biankai314.uk` are blocked by default.
- **No raw secrets in output**: Tokens are masked, zone IDs are not printed, raw API responses are not exposed.
- **No Web/Bot behavior change**: Web login and Bot owner gate remain unchanged.

## Commands

| Command | Mode | Description |
|---------|------|-------------|
| `nanobk setup flow` | Read-only | Full setup flow orchestration |
| `nanobk setup run` | Read-only | Alias for setup flow |
| `nanobk setup dns apply` | Plan-only | DNS creation plan |
| `nanobk setup dns apply --apply --confirm "..."` | Gated | DNS creation (requires confirm) |
| `nanobk setup cert plan` | Read-only | Certificate preflight |
| `nanobk setup cert issue` | Plan-only | Certificate issue plan |
| `nanobk setup cert issue --issue --confirm "..."` | Gated | Certificate issue (requires confirm) |
| `nanobk setup token rotate` | Plan-only | Token rotation plan |
| `nanobk setup token rotate --rotate --confirm "..."` | Gated | Token rotation (requires confirm) |

## Real VPS Acceptance Checklist

1. SSH into VPS
2. `git status` — working tree clean
3. `bash tests/v2.3.9-real-vps-acceptance.sh` — all checks pass
4. `nanobk setup flow --json` — inspect flow JSON
5. Verify no raw secrets printed in any output
6. Verify dangerous commands require copy-paste with exact confirmation
7. Verify production Worker remains protected (`nanok` plan-only shows `ready_to_rotate=false`)

## Known Non-Goals

- No automatic Worker deploy
- No automatic service reload/restart
- No Web/Bot live bridge mutation
- No release/tag in this stage
- No real DNS creation without explicit `--apply` + confirmation
- No real certificate request without explicit `--issue` + confirmation
- No real token rotation without explicit `--rotate` + confirmation
